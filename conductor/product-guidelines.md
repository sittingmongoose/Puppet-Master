# Product Guidelines

## Tone and Voice
- **Friendly & Encouraging:** The application communicates with the user in a supportive and conversational manner, helping to demystify complex "Vibecoding" workflows.
- **Supportive Clarity:** While friendly, instructions and feedback remain clear and actionable, ensuring the user always knows the status of their AI orchestration.

## Visual Identity
- **Preserve Established GUI:** Adhere strictly to the existing Rust/Iced GUI design. Any "tweaks" or additions must maintain the current aesthetic, layout, and user experience.
- **Consistency is Key:** Use existing widget styles, spacing, and color palettes to ensure new elements feel like an integral part of the original application.

## User Experience & Interaction
- **Pre-configured Logic:** Respect the existing orchestration and feedback systems. Automated actions should follow the established patterns for user notification and intervention.
- **Nudge over Interrupt:** When possible, provide non-blocking feedback and allow the user to steer the process according to the pre-defined "Puppet Master" workflows.
