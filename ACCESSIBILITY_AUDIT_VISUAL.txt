┌─────────────────────────────────────────────────────────────────────────────┐
│ FRAMEWORK LIMITATIONS (ICED)                                                │
└─────────────────────────────────────────────────────────────────────────────┘

  Iced is a NATIVE GUI framework (not web-based):
  
  ❌ NO ARIA Attributes
     - No aria-label, aria-describedby, role, etc.
     - Cannot add semantic metadata to widgets
     
  ❌ NO Semantic HTML
     - No <div>, <section>, <main>, <header>, etc.
     - No landmark navigation
     
  ❌ NO Live Regions
     - Status changes (e.g., "AI RESEARCHING...") are not announced
     - No aria-live, aria-atomic support
     
  ❌ LIMITED Screen Reader Support
     - Relies on OS-level accessibility APIs
     - Windows Narrator, macOS VoiceOver have basic support
     - NVDA/JAWS support varies by widget type
  
  ✅ WHAT ICED DOES PROVIDE:
     - Keyboard navigation (Tab, Shift+Tab)
     - Focus indicators (visible borders)
     - Text selection (via TextInput widget)
     - OS-level high contrast mode support

  ACTION: Document these limitations in an accessibility statement
