## Fixes applied

1. **Tauri Rust (all platforms):** `tauri::Error::Setup(String)` was replaced by `tauri::Error::Io(std::io::Error)` in `src-tauri/src/main.rs` because `SetupError` is not part of the public tauri API. Fixes compile error: “expected `SetupError`, found `String`”.

2. **Windows NSIS:** When `MAKENSIS_PATH` (or resolved path) contains spaces (e.g. `C:\Program Files (x86)\NSIS\makensis.exe`), the shell was splitting the command. In `scripts/build-installer.ts`, `resolveMakensisCommand()` now returns the path quoted when it contains a space so the NSIS step runs correctly.

3. **Linux Tauri deps:** Remote script `scripts/remote-build-linux.sh` installs apt packages (libwebkit2gtk-4.1-dev, pkg-config, etc.) when `SUDO_PASS` is set. Driver passes `SUDO_PASS_LINUX` so sudo apt-get runs non-interactively.

4. **Mac/Windows Rust:** Remote scripts install Rust via rustup when `cargo` is not found (Mac: curl \| sh; Windows: rustup-init.exe -y).

5. **Sync tarball:** Driver excludes `.claude` and `src-tauri/target` from the tarball to avoid tar errors on symlinks/special files.

## Scripts added

- `scripts/remote-build-linux.sh` – On Linux: optional apt deps (with SUDO_PASS), npm ci, build, gui:build, build:linux:tauri, cleanup .test-cache/.test-quota.
- `scripts/remote-build-macos.sh` – On macOS: Node/npm and Rust path or install, npm ci, build, gui:build, build:mac:tauri, cleanup.
- `scripts/remote-build-windows.ps1` – On Windows: WebView2 SDK, NSIS, Rust, npm ci, build, gui:build, build:win:tauri, cleanup.
- `scripts/remote-build-ssh-driver.sh` – Creates tarball, scp to each host, extracts, runs the platform script. Uses `SSH_PASS_LINUX`, `SSH_PASS_MAC`, `SSH_PASS_WIN`, optional `SUDO_PASS_LINUX`. Run from repo root: `./scripts/remote-build-ssh-driver.sh`.
