ROOT CAUSE IDENTIFIED
================================================================================

The actual root error (earlier in logs, not at the tail):

    LINK : fatal error LNK1181: cannot open input file 'WebView2Loader.dll.lib'

Why: GitHub Actions windows-latest runner does NOT include the WebView2 SDK
     (Software Development Kit) needed to COMPILE Tauri applications.
     
     ✅ Has: MSVC compiler, WebView2 Runtime (for browsing)
     ❌ Missing: WebView2 SDK headers/libraries (for compiling)

This is Windows-specific. macOS/Linux include their web rendering SDKs
in the standard development tools.

THE FIX (3 FILES CHANGED)
================================================================================

1. .github/workflows/build-installers.yml (NEW STEP ADDED)
   --------------------------------------------------------
   Location: Lines 63-77 (before "Install NSIS" step)
   
   Added:
   - name: Install WebView2 SDK (Windows for Tauri)
     if: runner.os == 'Windows'
     shell: pwsh
     run: |
       $sdkUrl = "https://www.nuget.org/api/v2/package/Microsoft.Web.WebView2/1.0.2592.51"
       # ... downloads and extracts SDK ...
       echo "WEBVIEW2_LIB_PATH=$libPath" | Out-File -FilePath $env:GITHUB_ENV
   
   Effect: Installs WebView2 SDK from official Microsoft NuGet package
           Adds ~8 seconds to build time
           Tauri's build.rs automatically detects WEBVIEW2_LIB_PATH


2. scripts/build-installer.ts (ERROR INSTEAD OF WARNING)

WHY THE BUILD APPEARED TO "SUCCEED" BEFORE
================================================================================

Old behavior:
1. ✅ TypeScript compiled
2. ❌ Tauri build failed (missing SDK)
3. ⚠️  Script logged warning, continued anyway
4. ✅ NSIS built installer WITHOUT Tauri app
5. ✅ CI marked job as "passed" (exit 0)

Result: Users got incomplete installer that opened browser instead of native app

New behavior:
1. ✅ TypeScript compiled
2. ❌ Tauri build fails (no SDK)
3. 🛑 Script throws error, stops immediately
4. ❌ CI marks job as "failed"
5. 🔄 Developer fixes SDK issue, retries

Result: Either complete installer with Tauri, or CI failure (not silent partial)
