FEATURE DETECTION:
- Scans decisions and Q&A history for feature keywords
- Weighted scoring: decisions (2.0-1.5x), answers (1.5x), questions (1.0x)
- Minimum 2 mentions required to qualify
- Returns top 5 highest-confidence features
- 11 categories: auth, api, payment, notifications, search, file-upload,
  realtime, chat, admin, reporting, analytics

PHASE NAMING:
- Standard phases: phase-01-scope-goals.md through phase-08-testing-verification.md
- Dynamic phases: phase-09-feature-auth.md, phase-10-feature-api.md, etc.
- Phase IDs: feature-{id} format (e.g., feature-auth, feature-payment)

INTEGRATION FLOW:
1. Complete Phase 8 (Testing & Verification)
2. Detect features from interview state
3. If features found:
   - Add each as dynamic phase via phase_manager.add_dynamic_phase()
   - Continue interview with Phase 9+ for each feature
   - Generate dedicated phase documents
4. Complete interview with all documents
